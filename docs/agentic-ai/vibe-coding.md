# Vibe Coding 实践（Coding Start）

“我不会写代码，也能做应用吗？”

在 Vibe Coding 的语境下，答案是：**可以从“会描述”开始**。你不一定要懂所有语法，但要学会把需求说清楚。

**短句总结：** **先会表达，再学细节。**

## 自然语言编程：Chat-to-Code 的基本方式

Chat-to-Code 的核心不是“让 AI 代写”，而是“让你更快迭代”。

一个实用的流程是：

1. **先描述目标**（功能、范围、限制）。
2. **要最小可用版本**（MVP）。
3. **再逐步加功能**。

例如：

```text
请用 JavaScript 写一个 3 秒倒计时的示例，输出到控制台，不要使用外部库。
```

可能得到这样的代码：

```javascript
let count = 3;
const timer = setInterval(() => {
  console.log(count);
  count -= 1;
  if (count === 0) {
    clearInterval(timer);
    console.log("start");
  }
}, 1000);
```

> 提醒：AI 生成的代码也可能有 bug，**务必运行并验证**。

**短句总结：** **最小可用 + 迭代** 才是高效节奏。

## 环境配置：让协作更顺滑

想把 Vibe Coding 用得顺手，环境配置很关键。

常见组合有：

- **VS Code + Copilot**：适合已有开发环境的人。
- **Cursor / Windsurf**：适合更强的 AI 编辑器体验。

你不需要一下子切换所有工具，先从“让 AI 进编辑器”开始，就已经是质变。

**短句总结：** **工具顺手** 才能长期坚持。

## 项目实战：从 0 到 1 的小应用

练习的目标不是做大项目，而是跑通流程。比如做一个“每日任务清单”：

1. **定义需求**：添加/完成/删除任务。
2. **拆分模块**：数据结构、UI、事件处理。
3. **先做最小版本**：能新增和显示即可。
4. **逐步增强**：加入持久化和样式。

你会发现，AI 可以帮你写代码，但**需求拆解仍然要你来做**。

**短句总结：** **你负责方向，AI 负责实现。**

## 小结

- Vibe Coding 的核心是“会表达”。
- 先做最小版本，再逐步扩展。
- 工具顺手是协作效率的放大器。

## 练习

1. 写一个“最小可用版本”的需求描述（不超过 80 字）。
2. 让 AI 写出一个函数后，自己手动改 1 处逻辑，并说明理由。
3. 设计一个 3 步小项目流程，并写出每一步的验收标准。
