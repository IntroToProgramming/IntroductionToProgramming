# Agentic AI：从对话到行动

2022 年末，ChatGPT 让很多人第一次直观感受到 AI 的能力。紧接着，新名词又一波接一波：Prompt Engineering、Agent、RAG、CoT……

你可能还在理解“大模型”，大家就已经在聊 **Agentic AI（智能体）** 了。会迷糊吗？很正常。

**直观地说**，这次变化只发生了一件事：

- 过去是“和 AI 聊天”，像在请教一个**顾问（Advisor）**。
- 现在是“和 AI 共事”，更像带一个能干的**实习生（Intern）**。

**一句话总结：** **从对话到行动**。

更严格一点说，Agentic AI 通常具备三类能力：**感知（Perception）**、**思考（Reasoning）**、**行动（Action）**，并且能通过**工具使用（Tool Use）**把想法变成结果。

接下来我们按这个思路，把它拆开讲清楚。

## 感知与表达：其实就是“好好说话”

你有没有带过新人？最怕的不是他不会做，而是你没把需求说清楚。和 AI 互动也是同样的道理。

大家常说的提示工程（Prompt Engineering），本质就是：**把意图说清楚**。

- 你想要什么结果？
- 允许或禁止什么？
- 有哪些背景信息？

**短句总结：** **清晰表达**，才会清晰执行。

现在的 Agent 还具备 **多模态（Multimodal）** 能力，能理解图片、文本，甚至视频。

* 想做海报？给它一张草图。
* 程序报错？截个图让它看。
* 想做视频？把分镜脚本扔给它。

别怕问得笨，也别怕试错。你越具体，它越靠谱。

## 思考与规划：像带新人一样带

当任务变复杂时，直接让 AI 开干，它可能也会“愣住”。

这时候，你要做两件事：

1. **给背景（Context）**：让它知道项目、目标、约束。
2. **拆任务（Decomposition）**：把大问题拆成一步一步。

这就引出了**思维链（Chain of Thought, CoT）**的思路：你不是在“让 AI 直接给答案”，而是在“让它先规划再执行”。

> 提醒：AI 可能会出现**幻觉（Hallucination）**。别急着相信结果，先让它解释思路，再自己核对。

**短句总结：** **先想后做**，更稳更准。

## 行动与构建：动口也能动手

最让人兴奋的，是 Agent 的**行动力**。它能调用工具、跑命令、修改文件，真正把事情做完。

在编程里，这被很多人叫作 **Vibe Coding（氛围编程）**：

- 你说“写个蓝色背景的网页”，它生成代码。
- 你说“跑一下看看”，它执行命令。
- 你说“报错了”，它读日志并修复。

**短句总结：** **描述逻辑** 就能开始编程。

更进一步，Agent 还能组合**技能（Skill）**，把多个动作串成一个小流程。这也是为什么它不只是“答题机器人”，而是“执行型助手”。

## 系统与治理：从单助手到小团队

当你熟练了，通常会发现：一个 Agent 不够用。

你需要：

- 一个负责查资料的
- 一个负责写初稿的
- 一个负责润色与校对的

这就是**工作流（Workflow）**。当多个 Agent 有分工、有协作、有反馈，就进入了**治理（Governance）**的范畴：

- 任务如何拆分？
- 谁来验收结果？
- 出错了怎么回滚？

**短句总结：** **会组织** 才能放大协作效率。

## 拒绝焦虑：Utilization is all you need

你可能会问：“模型这么多，我该学哪个？”

我们的建议是：**忘掉排名，拒绝焦虑。**

模型更新太快，今天第一名，明天可能就被超越。对学习者来说，更重要的是：**能用、好用、顺手用**。

* **随便聊聊**：X 上用 Grok，微信用元宝，微博用罗伯特。
* **办公场景**：在 Office 或 GitHub 里，**Copilot** 很顺手。
* **主力攻坚**：
  * 可访问国际网络时：**ChatGPT**、**Gemini**、**Claude** 依然强。
  * 依赖国内网络时：**豆包**、**Kimi**、**DeepSeek**、**Qwen（通义千问）** 也很好用。

**短句总结：** **顺手可用** 比排名更重要。

## 学习方式与工具

本教程的目标，是让你从“会用”走向“会协作”。

* **前置阶段（基础协同）**：
  * 准备一个你能访问到的聊天机器人网页即可。
  * 带着**开放心态**：多问、多试、多比较。

* **进阶阶段（Vibe Coding）**：
  * 推荐使用 **Cursor**、**Windsurf** 或 **VS Code + Copilot**。
  * 让 AI 直接进入你的编辑器，像同桌一样工作。

## 教程大纲

本教程分为两个阶段：前三部分面向所有用户，后两部分面向开发者与创造者。

### 第一部分：AI 协同基础（AI Essentials）
* **思维转变**：从“搜索”到“生成”，理解 LLM 的概率式输出。
* **高效沟通**：清晰指令、语境约束、多轮迭代。
* **场景实战**：写作、分析、决策辅助。

### 第二部分：多模态创作能力（Multimodal Creativity）
* **视觉创作**：图像生成工具的基本使用。
* **动态叙事**：视频生成与脚本表达。
* **工作流整合**：从文本到多媒体的创作链路。

### 第三部分：逻辑与推理（Reasoning & Logic）
* **深度思考**：用 CoT 拆解复杂问题。
* **准确性控制**：上下文管理与幻觉识别。
* **信息管理**：长文档与知识库应用。

--- *（以下章节面向开发者与进阶创造者）* ---

### 第四部分：Vibe Coding 实践（Coding Start）
* **自然语言编程**：Chat-to-Code 的基本模式。
* **环境配置**：VS Code + Copilot/Cursor 的协作环境。
* **项目实战**：从零构建并部署一个 Web 应用。

### 第五部分：智能体系统构建（Agentic System）
* **工具能力**：Tool Use 与终端/文件/API 的结合。
* **协议与标准**：理解 MCP（Model Context Protocol）。
* **自动化系统**：封装 Skill 与设计工作流。

## 小结

- **Agentic AI** 的核心变化，是从“对话”走向“行动”。
- **清晰表达 + 任务拆解** 是高效协作的关键。
- **工具使用与工作流** 让 AI 从助手变成团队成员。

## 练习

1. 用一句话描述一个你希望 AI 帮你完成的小任务，并补充 2 条限制条件。
2. 把一个“看起来很大”的目标拆成 3 个可执行的步骤。
3. 设想一个三人 AI 小团队，写下它们的分工与交接方式。
