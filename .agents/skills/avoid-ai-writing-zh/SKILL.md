# 避免 AI 写作：审读与重写

你正在审读和重写中文内容，目标是清理让文字像机器生成的 AI 痕迹。

这个中文版不是逐句翻译，而是把原技能的判断逻辑改成中文可直接使用的版本。

在这个工作空间里，它默认服务于虚构文学创作，所以除了通用中文表达问题，还要额外审理场面、对白、视角、章尾和人物声线。

## 模式

这份技能只有两种模式：

**`rewrite`**（默认，重写模式）  
标记 AI 痕迹，并直接重写文本，把问题修掉。

**`detect`**（只标记模式）  
只标记，不重写。适合以下情况：

- 作者只想看哪些地方被标出来，再自己决定怎么改
- 某些 AI 痕迹是有意保留的，未必都该删
- 你在审阅已发布内容、他人文本或不希望被改动的参考材料
- 你只想快速扫一遍，不想做完整重写

当用户说出 `detect`、`flag only`、`audit only`、`just flag`、`scan`、`只标记`、`只审不改`、`仅标记` 之类表达时，进入 `detect` 模式。未指定时，默认 `rewrite`。

---

在 **rewrite** 模式下，你的任务是：

1. **先审**：找出每一处 AI 痕迹，并引用具体文本
2. **再改**：给出清理后的版本，把 AI 痕迹去掉
3. **做简要说明**：概括你改了什么，以及为什么改

在 **detect** 模式下，你的任务是：

1. **先审**：找出每一处 AI 痕迹，并引用具体文本
2. **再判断**：说明哪些是明确问题，哪些可能是有意保留、在语境里也成立的写法

## 文学工作空间用法

- 如果任务来自某个 `projects/` 下的项目，优先考虑项目的语言限制，再用本技能去清 AI 痕迹。
- 本技能只负责“像不像 AI”，不负责替项目改定稿。项目里已经确认的声线、题材边界和章节任务，优先级高于这里的通用提醒。
- 在虚构文学里，能不能读懂不是第一标准。更关键的是：
  - 这句话是不是像这个视角会这样想
  - 这段对白是不是像这个人会这样说
  - 这段场面是不是先给动作和压力，再给判断
  - 章尾是不是留在物件、动作、半句对白或未完成压力上

## 需要删除或修正的内容

### 格式问题

- **破折号过多**：把 `—` 和 `--` 尽量改成逗号、句号、括号，或者直接重写。目标是零，硬上限是每 1,000 字最多一个。这个规则也适用于标题。
- **加粗过量**：大多数词不该加粗。每个大段最多保留一个强调点，或者干脆不用。
- **标题里放 emoji**：删掉。不要写成 `## 🚀 重点`。如果是社交短文，末尾可极少量使用，但不要夹在句子中间。
- **列表过密**：把一堆项目符号改成普通段落。只有真正适合列举的内容才用列表，比如步骤、参数、对比。
- **英文标题式大写**：混排英文标题不要每个词都大写。子标题优先用句式大小写；主标题也不要动不动把每个词首字母都抬起来。

### 句式问题

- **“不是 X，而是 Y”**：这类句式不要当默认模板。能直接说结论，就直接说结论。
- **空泛加强词**：删掉 `真正的`、`真的`、`事实上`、`坦白说`、`老实说`、`说清楚一点`、`值得一提`、`不得不说` 这类不提供信息的词。
- **空泛背书**：删掉 `值得阅读`、`值得关注`、`值得一看`、`值得深入`、`值得留意` 之类模板，改成说明“为什么重要”。
- **含糊缓和**：删掉 `也许`、`可能会`、`某种程度上`、`在某种意义上`、`可以说`、`大概` 这种只是在拖句子的词。
- **缺桥句**：段落之间要接得上。两个段落如果互换顺序也不影响阅读，说明桥接不够。
- **规则的三连**：少用固定的三分法和三连排比。可以用两项、四项，或者直接写成一句话。

这些词本身不坏，问题在于 AI 太爱堆。要看密度，不看单次出现。

| 词语 | 处理方式 |
|---|---|
| 重要 / 关键 / 必要 | 用具体事实替代一部分 |
| 有效 / 高效 | 说清楚怎么有效 |
| 创新 / 创造性 | 说明新在哪里 |
| 动态 / 变化 | 写出变化本身 |
| 多元 / 多样 | 直接列出有哪些不同 |
| 复杂 / 细致 | 说明复杂在哪 |
| 前所未有 / 空前 | 说明它和之前有什么不同 |
| 卓越 / 非凡 / 出色 | 用标准、数字或例子支撑 |
| 世界级 / 顶级 / 最佳 | 除非能比，否则别喊 |

### 模板短语

这些句式很像填空题。只要把空位换掉，整句还是一个味道，就要小心。

- `无论你是 X 还是 Y`：假装覆盖所有人，实际谁都没说清楚。
- `如果你正在……`：只有在确实面向这个人群时才留。
- `我最近有幸……`：社交文案和评论体里很常见，通常可以直接改成发生了什么。
- `在……的大背景下`：太像论文开头，能删就删。
- `当今这个……时代`：空泛，直接写具体情境。
- `让我们来看看`、`下面我们来拆解`：如果只是过渡，删掉。

### 过渡短语

下面这些词常常只是为了把话接上，实际上没有推动内容：

- `再者`
- `此外`
- `另外`
- `同时`
- `更重要的是`
- `与此同时`
- `在今天这个……语境里`
- `在一个……不断变化的时代里`
- `换句话说`
- `也就是说`
- `真正值得看的，是……`
- `最值得注意的是……`
- `更关键的是……`
- `总之`
- `综上`
- `最后`
- `概括来说`
- `一句话概括`
- `归根结底`
- `说到底`
- `说到`
- `值得一提的是`
- `不过`
- `话说回来`

原则很简单：能用语序、动作顺序、场景切换说清楚，就不要靠过渡词硬连。

### 结构问题

- **段落长度太整齐**：每段都差不多长，文本会像模板。要故意拉开差异。
- **开头太像引言**：不要先铺一大段背景再说重点。能先说结论就先说结论。
- **过度清洗**：把所有不规整的地方都磨平，反而更像 AI。保留一点自然断裂、短句、转折和语气变化。
- **三段式循环**：如果连续几段都是“背景 -> 行动 -> 感受 -> 总结”，要打散。
- **可以删一半仍成立**：如果一句话删掉一半意思还在，通常就该删。

### 何时该整段重写

如果一个文本同时满足下面三条，通常不要再做词语级修补，应该整段重写：

1. 词表命中已经超过 5 处，且分布在多个类别里。
2. 结构模式已经触发 3 类以上，比如概念先行、章尾总结、过渡词过密、段落节奏过于均匀。
3. 朗读起来还是像模板，像说明书，像自动生成稿，而不像人写的东西。

这时候，先把核心意思压成一句，再从场面、动作、关系或事实重新搭。

### 内容模式

下面这些是最重要的内容级判断。它们不只属于通用文案，也同样会污染小说、评论、摘要、项目说明和创作笔记。

- **意义膨胀**：把一件普通事写成“转折点”“里程碑”“历史性时刻”。如果事件本身不够大，就别先把它吹大。
- **无来源背书**：`专家认为`、`研究表明`、`很多人觉得`、`业内人士指出`。没有来源就不要拿权威口气顶上去。
- **宣传腔**：`充满活力`、`蓬勃发展`、`创新生态`、`未来可期`。如果这段不是广告，就别装广告。
- **伪分析**：把动作写成 `象征着`、`反映着`、`展示着` 这种连续解释，或者中文里一连串“体现着、象征着、说明着”。这类句子常常有内容，但没有新信息。
- **套路化挑战**：`尽管面临挑战，仍然蓬勃发展`、`在逆风中保持韧性`。先说具体挑战，再说具体应对，不然就是空句。
- **空泛范围**：`从……到……`、`从古代文明到现代创业`、`从大爆炸到暗物质`。看着很宽，其实什么都没说。
- **新颖性膨胀**：`他提出了一个概念`、`她发明了一个说法`、`这是一个没人注意到的问题`。大多数语境里，这类“新”都不新。
- **情绪标签代替展示**：`我很震惊`、`我很感动`、`一种复杂的情绪涌上心头`、`难以言喻的感觉`。如果是真的情绪，写动作、停顿、视线、呼吸和物件。
- **假平衡让步**：`虽然 X 很好，但 Y 仍然是问题`。如果两边都空，就只是姿态。要么具体让步，要么直接表态。
- **反问句开头**：`这意味着什么？`、`为什么这很重要？`、`接下来会怎样？`。如果你已经知道答案，就直接说答案。
- **括号式缓和**：`（更准确地说）`、`（尤其是）`、`（某种程度上）`。如果括号里的信息重要，就拆成独立句。
- **编号列表膨胀**：`三个关键点`、`五个理由`、`七件事`。只有真的有这么多并列项时，数字才成立。
- **推理链泄漏**：`让我们一步步来看`、`先考虑这个，再考虑那个`、`我的思路是`。这种内部推理痕迹不该直接出现在成稿里。
- **阿谀语气**：`好问题`、`你说得对`、`很有洞察力`。这是聊天界面的习惯，不是写作需要。
- **回答前复述**：`你问的是……`、`你是在问……`。直接答，不要先重复问题。
- **置信度词堆叠**：`值得注意的是`、`有趣的是`、`显然`、`当然`、`毋庸置疑`。如果事实本身够强，就别再给它戴情绪标签。
- **过度结构**：短文里塞太多标题、子标题和列点。结构要服务内容，不要替内容撑门面。
- **节奏过于均匀**：每句都差不多长，每段都差不多重。机器最爱这种节奏，读者通常也最容易累。
- **过度抛光**：把所有口气、停顿、碎裂感都磨掉，文字会变平。人写的东西本来就不会完全整齐。

| 阐明 / 说明 / 解释 | 直接把话说透 |
| 对照 / 对比 / 并置 | 说清楚比的是什么 |
| 变革 / 范式转移 / 结构性变化 | 说清楚具体变化 |
| 基石 / 关键支点 / 核心支撑 | 说清楚它起什么作用 |
| 至关重要 / 绝对重要 | 说清楚为什么重要 |
| 准备就绪 / 已经到位 | 说清楚哪一步完成了 |
| 萌发 / 萌芽 | 说清楚具体长出来什么 |
| 典型 / 典范 / 经典 | 说清楚为什么典型 |
| 总体 / 上位 / 宏观 | 如果只是转场，就删 |
| 底座 / 依托 / 依凭 | 说清楚谁支撑谁 |

#### 中文模板句

- `无论你是 X 还是 Y`：假装覆盖所有人，实际谁都没说清楚。
- `我最近有幸……`：社交文案里常见，通常可以直接改成发生了什么。
- `一小步，却是……的一大步`：把结论说满了，改成具体变化。
- `在……的大背景下`：太像论文开头，能删就删。
- `当今这个……时代`：空泛，直接写具体情境。
- `下面我们来看看`、`接下来我们拆一下`：如果只是过渡，删掉。
- `从……到……`：如果只是拼范围，就改成真正相关的范围。
- `如果你正在……`：只有确实面向这个人群时才留。

放到小说里，下面几类尤其危险：

- **概念先行**：先讲“这代表什么”“这意味着什么”，再写人物和场面。小说通常要先给人、物、动作和压力，再让意义自己长出来。
- **作者总结句**：段尾硬收成“这一刻他明白了”“这说明了一切”“从此故事不一样了”。这类句子会把叙事替换成判词。
- **对白书面腔**：人物开口像在写说明文、陈述意见、交代背景。对白要先服从身份、关系、体力和当下压力，再谈信息密度。
- **声线打平**：所有人物都说一个样。技能很容易把角色写成同一个“会解释的人”。
- **情绪标签代替展示**：`复杂的情绪`、`五味杂陈`、`难以言喻的感觉`、`一种说不清的压迫感`。如果情绪是真的，写动作、停顿、视线、手势、呼吸和物件。
- **章首背景灌输**：每一章都先解释设定、关系和历史，再往下走。小说应该先让场面站起来。
- **章尾模板化**：不是总收在“他意识到事情不对了”，就是总收在一个空泛钩子上。更好的收口通常是动作、物件、未完成的话、迟到的确认。
- **意象堆砌但不落地**：句子看着像文学，实际没有人物、动作、空间和时间压力。意象要服务场景，不要替场景站岗。
- **信息重复复述**：同一章里反复交代同一件事，只是换说法。小说不是提纲扩写。
- **统一口气的作者声**：旁白、人物、制度、解释都说成一种“稳重而正确”的声音。那通常是 AI 最容易露馅的地方。

## 输出格式

### rewrite 模式

按以下四部分返回：

**1. 问题清单**  
列出发现的每一处 AI 痕迹，并引用原文中的具体文本。

**2. 重写版本**  
给出完整重写后的内容。保留原意、原结构和必要事实，只改必须改的地方。

**3. 改动摘要**  
简要说明你做了哪些主要修改，不用逐词解释。

**4. 二次审读**  
重新读一遍第 2 部分的重写结果，找出第一次重写后仍然残留的 AI 痕迹，比如重复过渡、空泛加强、词表残留、结构太整齐等。把修正后的文本直接贴出来，并说明这次又改了什么。如果已经干净，就直接说清楚。

### detect 模式

按以下两部分返回：

**1. 问题清单**  
列出发现的每一处 AI 痕迹，并引用原文中的具体文本。按严重度分组：P0、P1、P2。

**2. 判断**  
对每一条标记说明它是明确问题，还是在当前语境里也可能成立的写法。并不是所有 AI 相关模式都该删，有些只是要看场景。若文本很干净，也要直接说明。

## 严重度分层

不是所有 AI 痕迹都一样。快速扫文时，按严重度优先处理。

### P0 - 可信度杀手，立刻修

- 回避式免责声明，比如“我目前无法访问实时信息”
- 聊天机器人口头禅，比如“希望这对你有帮助”“如果你愿意，我还可以……”
- 没有来源的空泛归因，比如“专家认为”“研究表明”
- 把平常事件吹成大事的意义膨胀，比如“里程碑”“转折点”

### P1 - 明显 AI 味，发布前修

- 词表里的高风险词
- 模板短语和空位填充句
- `让我们`、`下面我们` 之类的假协作开头
- 同一段里反复换同义词
- 套路化开场，比如“在……快速发展的今天”
- 加粗过多
- 破折号太多

### P2 - 风格层修整，有空再修

- 空泛结尾，比如“未来值得期待”“时间会给出答案”
- 规则的三连
- 段落长度太整齐
- 避免系动词，硬写成 `扮演着`、`充当着`、`提供着` 这类绕法
- 过渡词过密，比如 `此外`、`另外`、`同时`

做快速审读时，先看 P0 和 P1。完整审读再看三层。

## 语境档位

可以给一个可选的语境提示，用来调节严格程度。如果没有提示，就根据内容自动判断：

- 短文本 + 话题标签 / 提及，通常是 `linkedin`
- 有代码块、接口、架构说明，通常是 `technical-blog`
- 有称呼和融资、营收、增长语言，通常是 `investor-email`
- 有步骤说明、参数、说明文档结构，通常是 `docs`
- 没有明显信号，就默认 `blog`

### 档位定义

**`linkedin`**  
短社交文案。可以更短、更碎，但仍然要具体。

**`blog`**  
默认模式。长文标准，规则全开。

**`technical-blog`**  
技术文章、架构说明、接口文档。术语有正常技术意义时，不要因为“像营销词”就误杀。

**`investor-email`**  
高信任对象。宣传腔是最大风险，必须更紧。

**`docs`**  
文档、说明书、指南。清晰度优先于文风。

**`casual`**  
聊天、内部备注、快速回复。只抓最明显的问题。

### 小说档位

如果内容是虚构文学，优先在下面两个档位里判断，而不是机械套用 `blog`。

**`literary-fiction`**  
严肃文学、现实主义、实验叙事、慢节奏人物小说。这个档位更看重声线、留白、语气和场面压力，也更容忍句法弹性和局部碎片感。它不容忍的是解释欲、结论癖和修辞整齐到像模板。

**`web-fiction`**  
网络文学、连载小说、类型小说、强推进叙事。这个档位更看重节奏、冲突、钩子和信息清晰度，也更容忍直接表达、明确目标和高密度情节推进。它不容忍的是模板化开头、重复钩子和一章一个套路骨架。

如果没有明确提示，而内容明显是小说：

- 连载感强、章节钩子重、目标推进快，优先判成 `web-fiction`
- 语气更克制、留白更多、人物声音更重要，优先判成 `literary-fiction`

## 主题校准

目标是写出像人写的东西。直接、具体、有节奏。

五条原则：

1. **句长要变**：短句和长句混着来。碎句也可以。
2. **要具体**：把空泛判断换成数字、名字、日期、例子。
3. **要有声音**：如果语境允许，可以用第一人称，明确偏好和反应。
4. **别太中性**：如果文章本来就该表态，就直接表态。
5. **强调要赚到**：不要只说“这很重要”，要让读者看出为什么重要。

如果原文已经很好，就只做必要的删改，不要为了“清得更干净”反而把文字修死。替换表只是默认值，不是硬性命令；如果某个词在语境里就是最准确的，保留它。
